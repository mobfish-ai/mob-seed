# v3.5 Runtime Version - 运行时版本管理

> 状态: archived
> 创建时间: 2026-01-04
> 归档日期: 2026-01-04
> 发布版本: 3.5.0

## 概述

提供运行时版本管理能力，让用户了解当前使用的 mob-seed 版本，获知最新版本信息，并支持更新操作。

## 动机

当前 mob-seed 缺乏运行时版本提示机制：
1. 用户不清楚当前使用的版本
2. 无法知道是否有新版本可用
3. 更新需要手动执行命令，流程繁琐

## 目标

1. **版本展示**: 在关键操作时显示当前版本
2. **更新检查**: 定期检查 npm registry 获取最新版本
3. **更新提示**: 当有新版本时提供友好提示
4. **便捷更新**: 提供一键更新能力（可选）

## 预期产出

| 类型 | 描述 |
|------|------|
| 规格 | `runtime-version.fspec.md` - 版本管理功能规格 |
| 代码 | `lib/runtime/version.js` - 版本管理核心模块 |
| 测试 | `test/runtime/version.test.js` - 版本管理测试 |

## 风险与缓解

| 风险 | 缓解措施 |
|------|----------|
| 网络请求延迟影响用户体验 | 异步检查 + 缓存机制 |
| 离线环境无法检查 | 优雅降级，跳过版本检查 |
| 频繁检查浪费资源 | 设置检查间隔（如 24 小时） |
| 自动更新可能破坏项目 | 默认关闭，需显式启用 |
