# Insight Evaluate Prompt

> 用于 AI 辅助评估洞见时的提示词模板，引导用户进行辩证分析

---

## 评估框架

### 1. 核心观点提取

首先识别洞见的主要论点：

```
📌 核心观点分析

主要论点:
1. _______________
2. _______________
3. _______________

支持论据:
- _______________
- _______________
```

**提问引导**:
- 这个洞见的核心主张是什么？
- 作者用什么证据/案例支持这个主张？
- 有没有隐含的前提假设？

---

### 2. 适用性分析

评估洞见对当前项目/场景的适用性：

```
🎯 适用性评估

适用场景:
- _______________
- _______________

前提条件:
- _______________
- _______________

与当前项目的契合度:
[ ] 高度契合 - 可直接应用
[ ] 部分契合 - 需要调整
[ ] 低度契合 - 仅供参考
[ ] 不适用 - 场景不匹配
```

**提问引导**:
- 这个观点在什么场景下最有效？
- 我们的项目满足这些前提条件吗？
- 如果不完全适用，哪些部分可以借鉴？

---

### 3. 局限性思考

辩证分析洞见的局限：

```
⚠️ 局限性分析

已知局限:
- _______________
- _______________

潜在风险:
- _______________
- _______________

可能的反例:
- _______________

时效性考量:
[ ] 长期有效 - 基本原则不变
[ ] 中期有效 - 需定期复审
[ ] 短期有效 - 技术演进可能过时
```

**提问引导**:
- 这个观点有什么明显的局限性？
- 有没有场景下这个观点会失效？
- 随着技术发展，这个观点可能需要修正吗？
- 作者有没有承认某些局限？

---

### 4. 验证方案

如何验证洞见的有效性：

```
🧪 验证方案

验证方法:
1. _______________
2. _______________

成功指标:
- _______________
- _______________

试点范围:
- _______________
```

**提问引导**:
- 如何在小范围内验证这个观点？
- 什么样的结果可以证明观点有效/无效？
- 需要多长时间才能看到效果？

---

### 5. 采纳决策

基于分析做出决策：

```
📋 采纳决策

决策结果:
[ ] 全面采纳 - 直接应用到项目
[ ] 部分采纳 - 选择性应用
[ ] 试点验证 - 先小范围测试
[ ] 观望等待 - 等待更多验证
[ ] 不采纳 - 不适合当前场景

具体采纳内容:
✅ 采纳：
- _______________

⏸️ 观望：
- _______________

❌ 不采纳：
- _______________

后续行动:
- [ ] 创建相关变更提案
- [ ] 更新项目文档
- [ ] 安排试点验证
- [ ] 定期复审
```

## 状态转换指南

### evaluating → piloting

当满足以下条件时，可以转为试点状态：

- [x] 完成核心观点提取
- [x] 适用性分析显示有价值
- [x] 制定了验证方案
- [x] 确定了试点范围

```bash
/mob-seed:insight --update <id> piloting
```

### piloting → adopted

当满足以下条件时，可以正式采纳：

- [x] 试点验证通过
- [x] 效果符合预期
- [x] 无重大风险暴露

```bash
/mob-seed:insight --update <id> adopted
```

### piloting → partial

当满足以下条件时，可以部分采纳：

- [x] 部分观点验证通过
- [x] 其他部分不适用或有风险

```bash
/mob-seed:insight --update <id> partial
```

### * → rejected

当满足以下条件时，拒绝采纳：

- [x] 验证失败
- [x] 与项目需求不匹配
- [x] 风险过高

```bash
/mob-seed:insight --update <id> rejected
```

## 复审触发条件

以下情况需要重新评估已采纳的洞见：

1. **模型时代变更**: 新模型可能改变最佳实践
2. **技术演进**: 依赖的技术栈发生重大变化
3. **实践反馈**: 使用中发现问题
4. **时间触发**: 超过 6 个月未复审

使用以下命令检查需要复审的洞见：

```bash
/mob-seed:insight --review
```

---

## 辩证分析原则

1. **平衡看待**: 既看优点也看缺点
2. **场景敏感**: 考虑具体应用场景
3. **时效意识**: 考虑观点的时效性
4. **可验证性**: 确保决策可被验证
5. **迭代思维**: 允许观点随实践演化
